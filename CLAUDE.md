## Codebase Overview

Die Forward is a **social roguelite on Solana** (Phase 1 complete, Feb-Mar 2026) where players stake SOL, descend procedurally generated dungeons, and leave persistent corpses for others to discover. Built as a monorepo with Next.js web app (API + landing), Expo mobile app (primary gameplay), and Anchor smart contracts (escrow + run records). Active development; features: zone system, run modifiers, death milestones, inventory management, on-chain VRF randomness, Audius soundtrack integration. A three-rung staking ladder ("The Shift", phase 3b) sits alongside SOL staking: **Pale Coins** are an earned-only, never-purchasable in-game currency (`src/lib/coins.ts` / `mobile/lib/coins.ts`) that can be Coin-Bound at the Toll — death burns the stake into a pool-funded `coinPool`, escape returns it plus a bonus, and consecutive clears build a public **Binding Streak** (seal tiers) that a coin-bound death resets; every run writes a **run receipt** (`runReceipts` namespace) capturing `(runSeed, dayKey, dailyShiftEnabled, chosenModifierId)`; a `stakingPosture` admin setting (hidden/ritual/open) controls whether Blood-Bound (SOL) staking is shown at all. **Auth model (resolved, July 2026):** session/game requests carry a verified InstantDB token (`Authorization: Bearer <customToken>`, checked server-side via `verifyAuthToken` in `src/lib/auth-server.ts`); session routes derive the acting identity server-side from that token (never trust body-supplied `authId`/`walletAddress` for money ops — coin-mode `session/start` requires a verified token), and admin writes (`/api/admin/settings`, `/admin/bestiary`, `/admin/content`) are authenticated + admin-gated. Coin *earn* and player-row writes in `session/death`/`victory` are gated on a token-verified session flag (`authVerified`, stamped at start), closing the earn-side counterpart to the coin-deduct IDOR. `players.authId` is indexed with an enforced `string` type in the live InstantDB schema (`src/instant.schema.ts`, pulled/pushed via `instant-cli`) as of 2026-07-19, fixing a long-standing bug (since 2026-03-23) where `useCurrentPlayer()`'s email-derived, case-insensitive wallet lookup silently found no player for any wallet address containing an uppercase letter (79% of wallet accounts affected) — see `docs/KNOWN_ISSUES.md` "Recently Resolved". That `$ilike` lookup's introduction (2026-07-19) briefly exposed a LIKE-wildcard IDOR: `/api/auth/guest`'s `existingGuestId` re-auth path only checked `startsWith('guest-')`, so a crafted id containing `%`/`_` could mint a token whose derived `authId` matched other players' rows via the wildcard; fixed same-day with a strict UUID-shape regex on the server (`GUEST_ID_PATTERN`) plus a client-side charset guard in `useCurrentPlayer()` before any value reaches `$ilike`. Documented residuals (see spec Appendix E, §3.1 done-note): offline-local guests are client-authoritative by design, agent routes are session-id-gated but grant no coins, SOL payout address is still body-supplied (bound on-chain by the stake tx), `CRON_SECRET` must be set in prod before launch, and the `players` InstantDB perms still lack an ownership check — an *authenticated* client can directly write another player's `paleCoins` (grief-only on devnet; launch-blocking before coins gain value; needs a schema change since InstantDB `auth.id` ≠ our `authId`). Mature codebase with recent optimizations.

## Architecture

**Frontend:** Next.js 16 (`src/app/{page,api/*}`) serves API routes and landing page (deployed at `dieforward.com`); Expo Router (`mobile/app/*`) hosts mobile gameplay, shipped both as a native APK and as a web export at `play.dieforward.com` (Vercel project rooted at `mobile/`, configured in `mobile/vercel.json`). Same React Native code on both surfaces — see [`docs/DESIGN_SYSTEM.md`](docs/DESIGN_SYSTEM.md) for the shared palette and the `/design` reference page. **Shared state:** Mobile GameContext (`lib/GameContext.tsx`, ~1000 LOC) manages global game state; web/mobile sync via InstantDB. **Content engine:** `lib/content.ts` (~32k) defines bestiary, items, and `generateDungeonGraph` (dungeons are node DAGs — `ZoneNode`/`ZoneGraphLayout`, edges descend exactly one depth — with `currentRoom` as a canonical 1-based depth projection through GameContext, server routes, and the on-chain `u8`); all 5 zones now ship full branching graphs (20-23 nodes each) with client-only side chambers (`side: true`, item-gated via `gate: { item, consumes }`, Bone Dust reveals branch types before you commit); combat adds a Bait verb (`onBait` in `creature-rules.ts`) that forces the enemy's next intent AGGRESSIVE for a one-shot crit window; `lib/zone-loader.ts` handles the 5-zone system, validates zone graphs (`validateZoneGraph`), and is locale-aware (`ZONE_LOCALE_MAP`, per-zone English fallback) for zone-aware creature/item generation. Player-facing UI text is fully i18n-keyed and translated — 660-key catalogs across all 7 locales (`mobile/lib/locales/*.json`), plus locale-aware room/zone content (`lib/content.ts`'s `LOCALE_MAP` pattern for the 7 `content/*.json` room-content categories, mirroring `zone-loader.ts`'s `ZONE_LOCALE_MAP`). A player-facing language selector (`mobile/components/SettingsModal.tsx`, replacing the old audio-only `AudioSettingsModal`) auto-detects device locale on first launch, persists the choice (`mobile/lib/LocaleContext.tsx`), and switches the whole visible app live via a `subscribeLocale` pub-sub in `lib/i18n.ts` — no restart needed. A daily seeded world shift (`lib/world-shift.ts`, keyed on UTC day + zone) drives the modifier-choice pool and masks map edges/side doors each day, deterministically per zone; toggle via the `dailyShiftEnabled` game setting in `/admin`. A community layer (phase 4a) sits on top: a nightly server-receipted aggregation cron (`/api/game/shift`) writes a deny-by-default `worldShifts` InstantDB namespace (distinct-account thresholds, per-account caps, trailing-24h window — receipts-only, no client-supplied counts), which the mobile client merges additively over the seeded layer (`fetchCommunityShift`/`mergeShift`), degrading gracefully to the seeded-only layer offline or on fetch failure — surfaced as an apex-threat buff+bounty and non-UGC cursed/architect node markers. An A2 moderation gate (phase 4b, `src/lib/moderation.ts` — profanity/URL filter, trust-weighting by account age/staked history/wallet-auth, fail-closed on unknown authors; the `reports` namespace + authenticated `/api/moderation/report`) now gates all UGC rebroadcast: `worldShifts` also carries server-moderated `echoPhrases`/`architectEntries`, sourced only from trusted `runReceipts.finalMessage`, surfaced as Echo Husk *Repeating* recitals (combat) and Architect-wall `@nickname: words` inscriptions at the deadliest node, with a report button on the corpse surface. A shared `renderDispatch` pipeline (`mobile/lib/dispatch.ts`) feeds the home panel and zone-select; the same day's shift also drives a once-daily local-morning push whose recipient selection + text live in `src/lib/dispatch-fanout.ts` (`selectFanoutRecipients`/`renderPushText`), sent via `expo-server-sdk` (`src/app/api/game/dispatch/route.ts`, hourly cron) — STRICT F7 scarcity (push fires only on apex/cursed days, never on quiet/mask-only days, ≤1/day per `Player.lastDispatchDayKey`), diegetic optional opt-in (`Player.pushToken`/`timezone`/`notifOptIn`) with nothing gated on permission. **Progression:** `lib/modifiers.ts` (run modifiers), `lib/milestones.ts` (death milestones + perks). **Database:** InstantDB clients (`mobile/lib/instant.ts` for the app; `src/lib/db.ts` admin client for web API routes) manage Death, Corpse, Player schemas. **Auth:** Solana wallet signature verification (`src/app/api/auth/wallet/route.ts`), token-based session restore. **On-chain:** Anchor programs (`anchor-program/programs/*`) for escrow + run records; MagicBlock ephemeral rollups via `src/lib/magicblock.ts` (VRF integration). **Config:** Turbopack, path alias `@/*`, CORS globally applied to `/api/*`.

## Conventions

**Files:** Kebab-case for utils/contexts (`game-context.tsx`, `zone-loader.ts`); screens plain (`play.tsx`); types co-located in `{domain}.ts` files. **Imports:** Absolute `@/*` (web), relative (mobile); grouped: React → externals → types → utils → hooks. **Types:** PascalCase interfaces; string literal unions for enums. **Constants:** SCREAMING_SNAKE_CASE. **Components:** Functional, hooks-first; large state via context providers (GameContext), local state via useState. **Error handling:** Try-catch blocks with console.error; API routes return NextResponse with CORS headers. **Code style:** Deterministic RNG using seeded-random.ts; async/await for API calls; event handlers use useCallback to prevent re-renders.

## Testing

Jest is configured at both the root (`jest.config.js`, web) and in `mobile/` (`mobile/jest.config.js`, jest-expo preset). Mobile tests live in `mobile/lib/__tests__/` — 16 suites / 204 tests covering combat math, creature/item tags, synergies, signature rules, milestones, modifiers, dungeon generation, i18n, and more. Run mobile tests: `cd mobile && npm test`. Run web tests: `npm test` from the repo root. Test file convention: `*.test.ts(x)` co-located in `__tests__/` folders.

## Build & Run

**Install:** `npm install`. **Dev:** Web: `npm run dev` (localhost:3000); Mobile: `cd mobile && npm run start`, then `w`/`a`/`i`. **Build:** `npm run build` (web), `npm run build:web` (mobile export), `npm run build:android`/`build:ios` (EAS). **Local Android APK:** `cd mobile && npm run build:android:local` — flags `--prod` (release/signed/R8), `--metro` (live-reload), `--publish` (GitHub prerelease at `dev-<version>`); output `mobile/dist/`. See `mobile/BUILD_NOTES.md` and `mobile/docs/signing-secrets.md`. **Lint:** `npm run lint`. **Env:** Copy `.env.example` to `.env.local` at the repo root (web: `NEXT_PUBLIC_INSTANT_APP_ID`, `INSTANT_ADMIN_KEY`, MAGICBLOCK credentials, Solana RPC) and `mobile/.env.local` (mobile: `EXPO_PUBLIC_INSTANT_APP_ID` required, `EXPO_PUBLIC_SOLANA_RPC` / `EXPO_PUBLIC_API_URL` optional — inlined into the JS bundle at build time). **Package manager:** npm.

## Critical Paths

**Auth (CRITICAL):** `/src/app/api/auth/wallet/route.ts` — Solana signature verification; `/mobile/lib/auth.ts` — wallet sign-in, token storage, session restore. **Staking (CRITICAL):** `/mobile/app/stake.tsx` — SOL input UI; `/mobile/lib/solana/escrow.ts` — Anchor escrow instructions, buildStakeInstruction(), GAME_POOL_PDA. **Death records (CRITICAL):** `/mobile/lib/instant.ts` — recordDeath, discoverCorpse, recordTip; `/src/lib/db.ts` — Death/Corpse/Player schemas (web API). **On-chain (CRITICAL):** `/src/lib/magicblock.ts` — ephemeral rollup execution, VRF randomness; `/anchor-program/programs/run_record/` — run record program. **Game logic hub:** `/mobile/lib/GameContext.tsx` — central state machine, ~1000 lines; `/mobile/lib/zone-loader.ts` — zone-aware dungeon generation. **Files requiring extra caution:** stake.tsx, auth.ts, instant.ts, escrow.ts, magicblock.ts, wallet/route.ts.
